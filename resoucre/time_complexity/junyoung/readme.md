# 시간 복잡도란?
시간 복잡도란 알고리즘이 실행되는 데 걸리는 시간을 입력 크기에 대한 함수로 표현한 것입니다.

즉, 입력 크기(n)가 증가할 때 알고리즘의 수행시간이 어떻게 변하는지 설명합니다.

<br>

## 시간 복잡도의 표기법
일반적으로 Big-O 표기법을 사용하여 시간 복잡도를 표현합니다. 

Big-O 표기법은 최악의 경우를 기준으로 알고리즘이 얼마나 빠르게 실행될 수 있는지를 나타냅니다.

- Big-O(빅오)
  - Worst case
- Big-Θ(빅세타)
  - Average case
- Big-Ω(빅오메가)
  - Best case

<br>

### 왜 Big-O를 사용할까?
현실적인 성능을 보장하기 위해서 Big-O 표기법을 사용합니다.

알고리즘을 분석할 때, 최선과 평균적인 경우보다 최악의 경우 성능이 더 중요하기 때문에 시스템이 불안정한 상황에서도 성능을 보장할 수 있어야 합니다.

그래서 Big-O를 통해 최악의 성능을 파악하는 것이 필요하기 때문입니다.

<br>

### 그러면 우리는 왜 시간 복잡도를 알아야할까?
보통 코딩 테스트에서 cpu가 1초에 1억번에 연산을 한다고 정한게 국룰입니다.

문제가 2차원 배열의 탐색 문제가 나온다면 기본적으로 이중 for문을 사용해서 데이터를 찾을 수 있는데 이 때 시간 복잡도는 O(N^2)이 됩니다.

적은 데이터를 넣을 때는 문제가 되지 않겠지만 배열의 데이터가 10000x10000이 초과한다면 최악의 경우 cpu는 1초에 1억번 이상의 연산을 하기 때문에 시간초과가 될 수 있습니다.

그래서 우리는 시간 복잡도를 고려해서 최선의 알고리즘을 선택해서 문제를 풀어나가야 될지에 대한 고민을 해야됩니다.

<br>

### O(1) - 상수 시간
알고리즘이 입력 데이터의 크기와 상관없이 항상 일정한 시간 안에 실행되는 경우입니다. 이 복잡도를 가진 알고리즘은 입력 크기에 관계없이 항상 동일한 수의 단계를 거칩니다.

ex) 배열에 특정 인덱스의 요소에 접근하기

### O(logN) - 로그 시간
알고리즘이 입력 크기가 증가함에 따라 실행 시간이 로그 함수의 증가율로 증가합니다. 이런 알고리즘은 대개 문제의 크기를 각 단계에서 절반으로 줄여 나가며 해결합니다.

ex) 이진 탐색

### O(N) - 선형 시간
알고리즘의 실행 시간이 입력 데이터의 크기에 직접 비례하여 증가합니다. 입력 데이터를 한 번만 검사하거나 변형하는 작업이 여기에 해당합니다.

ex) 배열의 모든 요소 합계 계산

### O(NlogN) - 선형 로그 시간
알고리즘의 실행 시간이 입력 데이터 크기의 로그에 선형적으로 비례하여 증가합니다. 많은 효율적인 정렬 알고리즘이 이 복잡도를 가집니다.

ex) 병합 정렬, 퀵 정렬, 힙 정렬

### O(N^2) - 이차 시간
알고리즘의 실행 시간이 입력 데이터의 제곱에 비례하여 증가합니다. 간단한 중첩 반복문 구조에서 종종 볼 수 있습니다.

ex) 버블 정렬, 선택 정렬, 삽입 정렬

### O(2^N) - 지수 시간
알고리즘의 실행 시간이 입력 크기의 지수 함수로 증가합니다. 이러한 알고리즘은 입력 크기가 작을 때만 실용적입니다.

ex)피보나치 수열의 재귀적 계산

### O(!) - 팩토리얼 시간
알고리즘의 실행 시간이 입력 크기의 팩토리얼에 따라 증가합니다. 매우 비효율적이며, 일반적으로 매우 작은 입력 크기에서만 사용됩니다.

ex) 여행하는 판매원 문제(TSP)의 브루트 포스 해법

# 간단한 문제 풀이를 통한 시간 복잡도 계산

## 윤년 문제
연도가 주어졌을 때, 윤년이면 1, 아니면 0을 출력하는 프로그램을 작성하시오.

윤년은 연도가 4의 배수이면서, 100의 배수가 아닐 때 또는 400의 배수일 때이다.

예를 들어, 2012년은 4의 배수이면서 100의 배수가 아니라서 윤년이다. 1900년은 100의 배수이고 400의 배수는 아니기 때문에 윤년이 아니다. 하지만, 2000년은 400의 배수이기 때문에 윤년이다.

### 풀이와 시간 복잡도 계산
`year % 4 == 0 && (year % 100 != 0 || year % 400 == 0)`

4의 배수는 무조건 만족하며, 100의 배수가 아닐때 or 400의 배수일 때 를 조건으로 걸어서 풀었습니다.

각 연산은 모두 O(1) 입니다. 시간 복잡도는 O(1)이 됩니다.

## 과제 안 내신 분..?
X대학 M교수님은 프로그래밍 수업을 맡고 있다. 교실엔 학생이 30명이 있는데, 학생 명부엔 각 학생별로 1번부터 30번까지 출석번호가 붙어 있다.

교수님이 내준 특별과제를 28명이 제출했는데, 그 중에서 제출 안 한 학생 2명의 출석번호를 구하는 프로그램을 작성하시오.

### 문제 풀이와 시간 복잡도 계산
`studentNumber = studentNumber.filter{ $0 != number}`

배열을 돌면서 입련된 값을 제외시키기 때문에 시간 복잡도는 O(N)입니다.
