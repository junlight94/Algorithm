# 시간 복잡도란?
시간 복잡도란 알고리즘이 실행되는 데 걸리는 시간을 입력 크기에 대한 함수로 표현한 것입니다.

즉, 입력 크기(n)가 증가할 때 알고리즘의 수행시간이 어떻게 변하는지 설명합니다.

<br>

## 시간 복잡도의 표기법
일반적으로 Big-O 표기법을 사용하여 시간 복잡도를 표현합니다. 

Big-O 표기법은 최악의 경우를 기준으로 알고리즘이 얼마나 빠르게 실행될 수 있는지를 나타냅니다.

- Big-O(빅오)
  - Worst case
- Big-Θ(빅세타)
  - Average case
- Big-Ω(빅오메가)
  - Best case

<br>

### 왜 Big-O를 사용할까?
현실적인 성능을 보장하기 위해서 Big-O 표기법을 사용합니다.

알고리즘을 분석할 때, 최선과 평균적인 경우보다 최악의 경우 성능이 더 중요하기 때문에 시스템이 불안정한 상황에서도 성능을 보장할 수 있어야 합니다.

그래서 Big-O를 통해 최악의 성능을 파악하는 것이 필요하기 때문입니다.
